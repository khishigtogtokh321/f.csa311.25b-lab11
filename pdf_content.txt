Агуулга 
Tic-Tac-Toe (Монголчуудын нэрлэж заншсанаар Х/О) нь хоёр тоглогч ээлжлэн нүүдэг логик 
тоглоом юм. Ерөнхийдөө 3x3 хэмжээтэй хүснэгтэд тоглодог боловч хүснэгтийн хэмжээг илүү 
том байхаар шийдэж болно. Тоглогчид хоосон нүднүүдийг ээлжлэн өөрийн тэмдгээр (Х эсвэл 
О) бөглөх бөгөөд хэвтээ, босоо эсвэл диагональ чиглэлд гурван тэмдгийг түрүүлж 
дараалуулсан нь ялагч болдог.  
Энэхүү систем нь хоёр тоглогч нэг компьютер дээр хамт сууж тоглох боломжтой хувилбар юм.  
Технологийн шийдлийн хувьд: 
• Backend: Java хэл дээр суурилсан NanoHTTPD сервер 
• Frontend: React.js болон TypeScript (TS) 
Энэхүү клиент-сервер (Client-Server) архитектур нь веб программ хангамж боловсруулахад 
өргөн ашиглагддаг түгээмэл загвар юм. Хэдийгээр энд Java болон React-ийг хослуулсан боловч 
ерөнхийдөө backend болон frontend-ийн технологиудыг хэрхэн хослуулах нь маш өргөн 
сонголттой байдаг. 
Хүлээлгэн өгөх зүйлс 
• Тоглоомын "instructions" хэсэгт одоогийн нүүдэл хийж буй тоглогч болон тоглоомын

• Хэрэглэгч хийсэн нүүдлээ буцаах (Undo) боломжийг олгосон нэмэлт хөгжүүлэлт хийх 
• Дизайн болон харагдах байдлыг сайжруулах: CSS файл ашиглан дэлгэцийн өнгө төрх, 
интерфейсийн харагдах байдлыг сайжруулан өөрчлөх

үйлдлийг хэрэгжүүлнэ. 
 
Хэрэгжүүлэлт: Тоглоомын явцын түүхийг хадгалах "History" функцийг ашиглан олон

HTML/CSS тойм 
HTML (HyperText Markup Language) нь вэб хуудас бүтээх стандарт тэмдэглэгээний хэл юм. 
Бүтцийн хувьд XML-тэй төстэй. Харин CSS (Cascading Style Sheets) нь HTML баримтын 
загвар, дизайныг тодорхойлоход ашиглагддаг. Өөрөөр хэлбэл, CSS нь HTML элементүүд дэлгэц 
дээр хэрхэн харагдахыг (өнгө, хэмжээ, байршил г.м) зохицуулдаг. Хэрэв та эдгээр 
технологиудтай анх удаа танилцаж байгаа бол дараах онлайн эх сурвалжуудыг ашиглахыг 
зөвлөж байна. Үүнд: 
• HTML хичээл: w3schools.com/html 
• CSS хичээл: w3schools.com/css 
 
Nanohttpd тойм 
NanoHTTPD бол энгийн веб аппликейшн хөгжүүлэлтэд зориулагдсан, авсаархан (lightweight) 
Java сервер юм. Түүний гол давуу тал нь маш бага хэмжээтэй бөгөөд системийн ажиллагаанд 
нэмэлт ачаалал, хоцрогдол үүсгэдэггүй тул бэлэн байгаа программд суулгаж ашиглахад нэн 
тохиромжтой. 
Энэхүү Tic-Tac-Toe тоглоомын арын алба буюу Backend сервер нь NanoHTTPD дээр суурилсан. 
Техникийн дэлгэрэнгүй мэдээллийг NanoHTTPD зааварчилгаа холбоосоор орж үзнэ үү. 
(https://www.baeldung.com/nanohttpd). 
 
React.js тойм 
React.js нь интерактив, динамик хэрэглэгчийн интерфейс (UI) бүтээхэд зориулагдсан JavaScript 
сан (Library) юм. Үүнийг TypeScript-тэй хослуулан ашиглах бүрэн боломжтой. React-ийн 
үндсэн зарчим нь "Компонент" (Component) гэж нэрлэгдэх жижиг, бие даасан кодын 
хэсгүүдийг ашиглан нарийн төвөгтэй интерфейсийг хялбар аргаар угсрахад оршдог. 
Бид энэхүү төсөлд React.js-ийн хамгийн анхан шатны функцуудыг ашиглан тоглоомын 
фронтенд хэсгийг хөгжүүлсэн.

3-р үе шат: Орчин үеийн фреймворкууд ба React.js 
Өнөөг хүртэл ихэнх вебсайтууд 2-р шатны технологид тулгуурласаар ирсэн. Гэвч хуучны

холбож болох бөгөөд өгөгдөл өөрчлөгдөх үед фреймворк нь дэлгэцийн харагдацыг (UI) 
автоматаар шинэчилдэг болсон юм. 
 
Components 
Компонент нь React-ийн үндсэн суурь элемент юм. Компонент бүр дараах гурван зүйлийг 
өөртөө агуулдаг. Үүнд: 
 
• Бүтэц (HTML): Өгөгдлийг дэлгэцэнд хэрхэн дүрслэхийг тодорхойлох HTML 
элементүүд 
• Өгөгдөл (Data): Тухайн компоненттой холбогдсон өгөгдөл. Жишээ нь: Тоглоомын 
самбарыг харуулах компонент нь самбарын утгуудыг агуулсан өгөгдөлтэй холбогдсон 
байна. 
• Логик (Logic): Өгөгдөл болон харагдац хэрхэн динамикаар өөрчлөгдөх, хэрэглэгчийн 
үйлдэлд (товчлуур дарах гэх мэт) хэрхэн хариу үйлдэл үзүүлэхийг зохицуулах код 
 
Иймд бид веб аппликейшнийг жижиг хэсгүүдэд (компонентуудад) хувааж, хэсэг бүрийг 
өөрийн гэсэн өгөгдөл, логик, харагдацтайгаар бие даасан байдлаар хөгжүүлэх боломжтой.

Props болон State-ийн ялгаа 
React компонент бүрт дараах хоёр гол объект байдаг. Үүнд: 
• Props (Properties): Энэ нь эцэг (parent) компонентоос дамжуулсан өгөгдөл юм. Жишээ 
нь: Welcome компонентод name гэсэн утгыг дамжуулбал бид this.props.name ашиглан 
тэрхүү нэрэнд хандана. Props-ийг хүлээн авсан компонент өөрөө өөрчлөх ёсгүй, зөвхөн 
дамжуулсан эцэг компонент нь өөрчлөх эрхтэй. 
• State (Төлөв): Энэ нь компонентын дотоод өгөгдөл бөгөөд тухайн компонент өөрөө 
удирддаг. State-ийг шинэчлэхийн тулд 
setState() функцийг ашиглана. Утга 
шинэчлэгдэх бүрд React дэлгэцийг (UI) автоматаар дахин зурдаг. 
• 
render() функц: Компонент хэрхэн харагдахыг тодорхойлдог цорын ганц заавал байх 
ёстой функц юм. 
JSX-ийн тухай 
React дээр бид JSX (JavaScript XML) ашиглан HTML-ийг JavaScript эсвэл TypeScript код дотор 
шууд бичдэг. Энэ нь кодыг илүү ойлгомжтой, динамик болгодог. Жишээ нь, таг (tag) дотор 
{this.props.name} гэж бичвэл нэр өөрчлөгдөх үед React харагдах байдлыг автоматаар 
шинэчлэх болно. 
 
Зарим чухал тэмдэглэл: 
• Props: Компонент дотроос өөрчлөх боломжгүй (Read-only) 
• State: Компонент дотор өөрчлөх боломжтой (Mutable via setState) 
• Constructor: Заавал байх шаардлагагүй, харин render() функц заавал байх ёстой.

үндсэндээ фронтенд (Frontend) талаас гарч буй бүх харьцангуй замын хүсэлтүүдийг 
package.json файлд зааж өгсөн үндсэн URL (Base URL) руу чиглүүлэх үүргийг гүйцэтгэдэг.
